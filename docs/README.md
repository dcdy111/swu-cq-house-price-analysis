# Docs 目录说明

当前 `docs/` 已按“可直接交人、可直接查证”的方式整理：

- `deliverables/`
  最终提交材料、答辩用文档和对外交付件。
- `evidence/`
  页面截图、接口验收 JSON、增量更新证据、Playwright 验证产物。
- `reports/`
  测试报告、自动化验收报告、整体验收总结。
- `references/`
  原始需求文档、开发约束文档、数据源说明等参考资料。
- `notes/`
  迭代方案、清理记录、阶段性评估笔记。

建议查看顺序：

1. `reports/全测试报告_20260701_最终版.md`
2. `evidence/acceptance_evidence.json`
3. `deliverables/`
4. `references/`

如果后续继续补材料，请保持这个目录结构，不要再把报告和参考文档散落到 `docs/` 根目录。
