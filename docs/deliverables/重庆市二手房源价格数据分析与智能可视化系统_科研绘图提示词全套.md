# 重庆市二手房源价格数据分析与智能可视化系统：科研绘图提示词全套

## 一、使用方式

推荐把每一节的提示词单独发给 GPT，并要求它**优先输出可运行的 Python + Graphviz 代码**，同时导出 SVG 和 300 dpi PNG。与直接使用文生图相比，Graphviz 更容易保证中文准确、箭头关系正确、字体统一，也便于后续在论文和 PPT 中修改。

所有图片都遵守以下要求：

- 图片内部不写“图 3-1”“图 X”或论文式图号；图名和说明由正文单独添加。
- 不使用 AI 生成式插画、发光渐变、3D 服务器、卡通人物、装饰性大图标。
- 只画系统真实存在的功能，不画区块链、知识图谱、微服务、Docker、实时大数据平台等未实现内容。
- “公网部署”“团队成员姓名”没有事实证据时，只能画为待实施方案或占位模板，不能画成已完成事实。
- 系统中的价格统一写“挂牌价/报价”，不得写“成交价”。

## 二、项目故事线

建议说明书和答辩统一使用下面这条故事线：

1. **问题背景**：重庆二手房挂牌信息来源分散、字段不统一、网页内容会变化，直接分析容易出现重复、缺失和口径不一致。
2. **数据工程**：使用低并发多线程采集，将网页数据解析为统一字段，通过 fingerprint 去重，并用快照记录价格变化。
3. **可信底座**：MySQL 统一保存房源、快照、任务、日志、质量、模型和 Agent 证据；最终数据达到 100,897 条有效房源。
4. **质量治理**：从完整性、唯一性、一致性、及时性、有效性、可核验性六个维度评价数据，异常记录保留并进入复核。
5. **分析建模**：使用 EDA、回归模型对比、KMeans 分层和 IsolationForest 异常检测，结论同时展示样本量和误差指标。
6. **可视化与问数**：React + ECharts 展示区县、价格、面积、户型和模型结果；DeepSeek 只能通过白名单工具获取证据。
7. **系统价值**：形成“采集—治理—分析—展示—问数—反馈补采”的可追溯闭环，而不是只生成一次性静态图。

### 9. 分析建模流程图

正文图名：**分析建模流程图**

```text
生成横向机器学习流程图，分成“样本准备—并行分析—结果落库—页面解释”四段。
```
